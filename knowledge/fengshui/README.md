# Fengshui And Direction Knowledge Card

source_id: `SRC-PROJECT-FENGSHUI-DIRECTION-SYNTHESIS`
classical_source_id: `SRC-FENGSHUI-ZHAIJING`

这个模块用于报告里的方位、空间、城市、办公位置、居住习惯和轻量开运建议。它是辅助层，不是主判断层。

## 来源边界

- `SRC-FENGSHUI-ZHAIJING` 登记《宅经》（旧题《黄帝宅经》）作为风水方位、阴阳宅、二十四路和宅居术语的传统入口。
- 这个来源当前是 `candidate`，只用于 provenance、术语边界和传统语境说明，不把古宅法直接套到现代住宅、办公室或城市选择。
- 报告里的风水方位建议仍以 `SRC-PROJECT-FENGSHUI-DIRECTION-SYNTHESIS` 为执行标准：现实目标、空间信息、可撤回调整和复盘指标优先。
- 没有现场勘测、罗盘坐向、户型图和长期反馈时，不把任何传统方位写成坐向吉凶、发财破财或灾祸定论。

## 使用范围

- 可写：五行补偏、空间习惯、办公/学习位置、出行方向、城市气质、居住环境对节奏和情绪的影响。
- 可写：基于八字喜忌、紫微宫位主题、西占轴线和现实生活目标，给出低风险的环境调整建议。
- 可写：城市选择里的气候、行业、客户密度、交通、生活成本和资源网络，再辅以五行象意。
- 不写：没有现场勘测、罗盘坐向、户型图和长期居住反馈时，不做传统阳宅风水强断。
- 不写：不承诺“换方向必发财”“摆物必转运”“某方位一定有灾”。

## 证据顺序

1. 现实目标：读者要改善的是睡眠、专注、客户、合作、身体恢复、输出效率还是城市发展。
2. 结构化 facts：八字五行、喜忌倾向、大运流年、紫微宫位和西占轴线只作象意锚点。
3. 生活场景：办公桌、卧室、通勤、城市气候、客户来源和社交半径。
4. 低风险建议：先写可观察、可撤回、低成本的调整。
5. 校准反馈：建议执行 2-4 周后再看睡眠、效率、情绪、客户响应或合作摩擦是否变化。

## 证据等级

- `level_0_no_fengshui`: 用户没有问城市、方位、空间或开运，正文不主动加风水段落。
- `level_1_lifestyle`: 只有当前城市、办公/居住困扰或生活目标时，只写空间习惯和现实条件，不写坐向吉凶。
- `level_2_direction_preference`: 有出生资料、当前/候选城市、办公或居住场景时，可写“更适合优先试的方向/环境”，但必须给现实理由和观察指标。
- `level_3_site_informed`: 有户型图、罗盘坐向、楼层、采光、噪音、潮湿、房间功能和居住反馈时，才可讨论更细的空间布置，但仍不写绝对发财、破财或灾祸。
- `level_4_professional_survey`: 需要现场勘测、罗盘复测和长期反馈；本项目默认不冒充这一层。

## 方位与空间主题

- 城市方位：适合用于候选城市、迁移、客户来源和事业重心。先比较行业资源、客户密度、成本、家庭生活、通勤和现金流，再用五行气候或命理象意解释为什么某类城市更顺手。
- 办公方位：适合用于专注、表达、客户沟通和协作摩擦。优先写桌面动线、背后安全感、采光、噪音、屏幕朝向、会议动线和可被打扰程度。
- 居住方位：适合用于睡眠、恢复、家庭承载和情绪稳定。优先写床位稳定感、潮湿、光照、噪音、储物压力、通风和家庭成员使用冲突。
- 出行方向：只能用于短期行动偏好和心理启动，不替代择日、投资、医疗或重大迁移决定。
- 开运辅助：优先写“减少消耗、增加复盘、提高输出、降低干扰”的行为建议；颜色、摆件、饰品只能作为低权重偏好，不作为核心处方。

## 八方象意速查

此表只用于报告里的象意翻译，不等于罗盘风水定论。没有罗盘坐向、户型图和现场反馈时，不能把下面任何一项写成“吉凶方”。

| 方位 | 五行象意 | 可用于解释的现实主题 | 低风险写法 |
| --- | --- | --- | --- |
| 东 | 木 | 生长、学习、启动、内容更新、年轻客户 | 适合试更有生发感、日程更规律、方便输出的位置 |
| 东南 | 木中带风 | 传播、人脉、流动、合作入口、内容扩散 | 适合试更利于沟通、客户触达和轻社交的动线 |
| 南 | 火 | 曝光、表达、名声、审美、销售呈现 | 适合增加可见度、展示面和稳定表达节奏 |
| 西南 | 土 | 承载、家庭、团队照顾、稳定生活 | 适合处理家庭协同、后勤承载和长期稳定感 |
| 西 | 金 | 规则、合同、审美、收敛、成果验收 | 适合强化交付标准、合同边界和结果复盘 |
| 西北 | 金 | 权威、决策、资源、领导、制度 | 适合观察决策链、贵人资源和组织支持度 |
| 北 | 水 | 信息、流动、休息、研究、远方机会 | 适合补沟通、学习、复盘、远程连接和休息质量 |
| 东北 | 土 | 停顿、储备、学习转向、门槛 | 适合做知识沉淀、资格门槛和阶段性整理 |
| 中宫 | 土 | 平衡、承压、整体秩序、家庭或组织中心 | 适合看空间是否被杂物、冲突或过多功能压住 |

使用规则：

- 先问现实目标，再选方位象意；不能先选“吉方”再套现实。
- 单盘只可写“更适合试哪类环境”，合盘或团队盘才可写“谁适合坐前台、谁适合守交付、谁适合谈客户”这类分工建议。
- 方位象意必须和至少一条命盘/现实证据同向，才可进入正文；否则写成自查问题。
- 如果用户给了户型图或罗盘坐向，但没有现场复测，本项目仍按 `level_3_site_informed` 写，不升到专业勘测。

## 场景写法

### 城市和迁移

城市判断先按现实商业条件排序，再用方位象意解释。报告里可以写：

- 客户和行业资源优先：客户在哪里、供应链在哪里、同行密度如何、现金流是否承受得住。
- 生活承载优先：房租、通勤、家庭照顾、气候适应、医疗教育和恢复空间。
- 命理辅助：五行气候、迁移宫/大运/流年触发、八字喜忌和个人工作方式是否同向。
- 可观察指标：3 个月内客户响应、通勤疲劳、睡眠恢复、社交半径、合作推进速度。

不要写“去南方必火”“去北方必发财”。可以写“如果现实条件相近，更值得优先试节奏更开放、客户更集中、表达机会更多的城市；这类环境比单纯换方向更关键”。

### 办公和学习

办公位先看干扰、背后安全感和输出节奏。报告里优先建议：

- 背后减少频繁走动，屏幕不被路过的人持续看到。
- 桌面只保留正在推进的项目，减少多任务视觉噪音。
- 需要表达、销售、内容的人，增加展示面和固定输出时段。
- 需要研究、财务、法务、技术的人，优先安静、边界清楚、可长时间深度工作的区域。
- 合伙团队可以按角色分工：对外客户位、内部交付位、复盘决策位不要长期混在一起。

### 居住和睡眠

居住建议只做低成本、可撤回调整：

- 先处理噪音、潮湿、光照、通风、储物压力和家庭成员使用冲突。
- 睡眠问题先看床位是否被通道、门口、强光、噪音或工作物品持续打扰。
- 恢复差的人，卧室不要兼太多工作、直播、社交或杂物功能。
- 家庭承载重的人，先减少公共区冲突和物品堆积，再谈摆件颜色。

### 开运和物件

开运辅助不以摆件为主。优先级是：

1. 调整消耗源：噪音、通勤、杂物、被打扰、无边界协作。
2. 建立稳定动作：睡眠、运动、复盘、输出、客户跟进。
3. 用颜色、植物、灯光或物件作为提醒物，而不是当作“法器”。
4. 任何花钱、搬家、装修、买大件的建议都必须降级为现实决策，不用命理直接推动。

## 常见降级

- 缺罗盘坐向：不写“某坐某向吉凶”，只写环境感受和使用方式。
- 缺户型图：不写房间级布局强断，只写可移动、可撤回的调整。
- 缺长期反馈：不把一次体验写成定论，只建议观察 2-4 周。
- 缺现实目标：先问要改善什么；不要为了显得丰富而硬加方位。
- 涉及搬家、买房、合伙选址或大额投入：必须写现实成本和商业条件优先，命理方位只作参考层。

## 写作边界

- 方位建议必须写成“更适合试”“可以优先考虑”“不建议长期压在某种环境里”，不能写成绝对命令。
- 城市判断先商业后命理；方位象意只能辅助解释，不能替代市场和现金流。
- 开运建议优先写行为和环境，不写昂贵物品、迷信摆件或恐吓式化解。
- 如果没有出生地、居住地、户型、坐向或执行反馈，只能写一般性空间偏好和校准问题。
- 高风险领域仍降级：医疗、法律、投资、婚恋重大决定不能用方位建议替代专业判断。

## 报告写法

- 先写证据等级：例如“这不是现场风水勘测，只能按命盘倾向、城市条件和当前空间问题给低风险建议”。
- 再写现实问题：例如睡眠浅、工位被打断、客户距离远、城市资源不匹配。
- 再写可试方向：例如“优先试更安静、背后有靠、采光稳定、减少穿堂干扰的位置”，不要只写东南西北。
- 最后写复盘指标：睡眠时长、起床状态、专注时段、客户响应、沟通摩擦、通勤疲劳、家庭冲突。
- 如果用户明确问“我适合哪个方向/城市/工位”，回答顺序固定为：结论排序、现实理由、命理辅助、低成本试法、复盘指标。
- 如果证据不足，先给缺口清单：当前城市/候选城市、目标主题、户型或工位信息、可接受成本、观察周期。

可用句式：

- “如果只按现有资料看，这里只能给低风险空间建议，不做坐向吉凶强断。”
- “更值得试的是让办公位减少背后干扰、固定输出时间，而不是花钱买摆件。”
- “城市选择先看客户和现金流，命理象意只说明哪类气候、节奏和资源场更容易让她发挥。”
- “这条建议需要 2-4 周复盘；如果睡眠、专注和客户响应没有变化，就不能把它当成有效规则。”

## 复盘采集点

- 调整后睡眠、专注、效率、沟通或客户响应是否有可描述变化。
- 哪条建议太玄、太泛或无法执行。
- 哪条建议其实来自现实生活结构，而不是方位象意。
- 城市/方位判断有没有被后续现实验证或推翻。
- 适用限制是什么：行业、预算、房型、通勤、家庭成员、健康状态或季节。
- 是否有户型图、罗盘坐向或现场反馈；如果没有，报告是否明确降级。
- 建议执行成本是否合理；有没有因为建议太麻烦、太贵或不可撤回导致无法验证。

## 反馈记录最小字段

风水方位类反馈必须先留在 run-local，不直接晋升知识库。记录时至少保留这些去隐私字段：

- `original_goal`: 想改善的现实目标，例如睡眠、专注、客户响应、团队沟通、城市试点。
- `evidence_level`: 使用本卡的 `level_1_lifestyle`、`level_2_direction_preference` 或 `level_3_site_informed`，不要事后抬高证据等级。
- `suggestion_type`: 城市 / 办公位 / 居住空间 / 出行方向 / 开运辅助。
- `suggestion_summary`: 去掉姓名、城市、地址和具体项目后的建议摘要。
- `actual_change`: 2-4 周后可描述的变化；没有变化也要记录。
- `likely_driver`: 更像来自现实环境调整、生活习惯、商业资源、执行动作，还是方位象意。
- `misread_or_limit`: 哪一句太玄、太泛、太贵、不可执行，或因为缺户型/坐向必须降级。
- `next_rule_candidate`: 这条反馈是否能改进 `knowledge/fengshui/README.md`、资料收集表或报告模板。

不要把单次反馈写成“某方位必然有效”。至少要有两类信息同时成立，才考虑晋升为知识库候选：一是建议和实际变化之间有可观察链条，二是已经说明现实驱动和方位象意的边界。

## 候选复盘命令

当读者给出执行反馈，先在 `<RUN_DIR>/calibration/dialogue/` 记录原始追问和回答口径，再生成候选复盘：

```powershell
python -X utf8 scripts\create_case_retrospective_candidate.py --manifest <RUN_DIR>\case_manifest.json --slug fengshui-direction-feedback --title "去隐私风水方位执行反馈复盘" --domain fengshui --evidence-summary "只写去隐私后的目标、建议、执行变化和降级边界" --target-artifact knowledge/fengshui/README.md
```

人工确认后才可晋升：

```powershell
python -X utf8 scripts\promote_case_retrospective.py --candidate <RUN_DIR>\retrospectives\CR-YYYYMMDD-fengshui-direction-feedback.candidate.json --approved-by <APPROVER> --dry-run
python -X utf8 scripts\promote_case_retrospective.py --candidate <RUN_DIR>\retrospectives\CR-YYYYMMDD-fengshui-direction-feedback.candidate.json --approved-by <APPROVER>
python -X utf8 scripts\audit_case_retrospectives.py
python -X utf8 scripts\audit_knowledge_coverage.py
```
