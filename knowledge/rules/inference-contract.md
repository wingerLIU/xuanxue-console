# Inference Contract

这个合同规定：排盘事实怎么变成付费报告里的判断。目标不是让文章显得玄，而是让每个判断有证据等级、适用范围和可校准入口。

## 证据等级

1. `calculated_fact`: 脚本输出事实，例如四柱、日主、大运、命宫、行星落座、上升、天顶。
2. `classical_rule`: 来自经典或流派规则，但已经写清来源、适用范围和限制。
3. `modern_synthesis`: 对多个体系的现代综合解释，例如“判断力需要产品化”。
4. `case_calibration`: 通过客户过去经历确认或待确认的判断。
5. `practical_advice`: 现实建议，例如合同、预算、沟通、作息、作品集。

## 证据优先级

1. 输入资料和脚本 JSON 优先于模型记忆。
2. 时间敏感性检查优先于漂亮断语。
3. 多体系交集优先于单点象意。
4. 已校准事实优先于未校准推测。
5. 风险边界优先于营销吸引力。

## Claim Packet

报告中的核心判断，写作前应能在脑中形成一个最小证据包：

```text
claim: 你在什么议题上更像什么结构
evidence:
  - calculated_fact: 来自哪个 JSON 事实
  - rule_card: 来自哪个知识卡
  - synthesis: 为什么这些证据可以合起来读
limits: 哪些情况下这个判断会失效
calibration: 读者可以用什么经历验证
advice: 能马上做什么
```

正文不需要把这个包逐字写出，但不能没有这个包。

## 强弱表达

- 可以较强表达：脚本事实稳定、多个体系同向、时间不敏感、风险低、可用现实经历验证。
- 只能弱表达：单一星曜/神煞/相位、出生时间不稳、流派分歧大、涉及健康/投资/婚姻/法律。
- 必须校准：过去年份事件、家庭关系细节、亲密关系状态、具体职业路径、重大财务节点。

## 跨体系合参

- 八字优先解释季节、五行、十神、运势节奏和现实承载方式。
- 紫微优先解释宫位角色、人生领域、四化触发和阶段课题。
- 西占优先解释心理语言、关系表达、第一印象、职业形象和时间敏感轴线。
- MBTI 只能作为行为语言层，不作为命理证据。
- 占事类模块只回答当前问题，不覆盖出生盘结构。

当体系冲突时，不强行合并。写法应是：“这里有两股力量，一股是……另一股是……现实里可能表现为……需要用……校准。”

## 写作要求

- 开头可以有断言，但后文必须给证据链。
- 不把 `modern_synthesis` 冒充为古籍定论。
- 没有客户校准前，过去事件写“可能、倾向、可校准问题”。
- 技术证据附录只列事实，不加入新的情绪化判断。
- 每次 case 结束，如果发现重复问题，先写 run-local retrospective；人工确认后才能提炼到全局知识库。
- 本人付费报告默认用第二人称“你”；第三方报告才用“他/她/对方”。

## 禁止事项

- 不由模型手算四柱、大运、星曜、宫位或行星落座。
- 不预测死亡。
- 不诊断疾病。
- 不给投资买卖点。
- 不替客户做结婚、离婚、诉讼、辞职等重大决定。
- 不用恐吓话术引导复购。
- 不把客户截图、出生资料、本机路径或交付稿写入全局知识库。
