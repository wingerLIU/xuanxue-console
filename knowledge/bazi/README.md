# Bazi Rules

八字知识卡只写可复用规则，不写客户案例原文。

推荐每张规则卡包含：

- `rule`: 规则或判断框架。
- `source`: 来源索引条目，例如 `source-index.md#八字候选来源`。
- `scope`: 适用范围。
- `limits`: 不适用或容易误判的情况。
- `modern_translation`: 现代现实语言怎么写。
- `report_usage`: 在付费长文里如何落地。

当前默认边界：

- 四柱、大运、流年、十神、五行数量必须来自脚本 JSON。
- 调候、格局、喜忌是解释框架，不单独制造结论。
- 神煞只作辅助象意，不覆盖格局、十神、大运和现实校准。

## 规则卡

- `foundations.md`: 结构优先、月令、旺衰、调候/扶抑/通关、大运流年。
- `classical-anchors.md`: 已登记典籍入口提炼出的方法锚点。
- `shenfeng-anchors.md`: 《神峰通考》病药、盖头、动静、枯旺等方法边界。
- `shenfeng-methods.md`: 《神峰通考》方法锚点的报告推理步骤、反例和落地模板。
- `ten-gods.md`: 十神现实语言、风险、报告写法和常见组合。
- `structure-and-flow.md`: 地支关系、神煞、大运流年和未来趋势边界。
