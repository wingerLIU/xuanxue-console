# Bazi Structure And Flow

source_id: `SRC-BAZI-PROJECT-SYNTHESIS`

这个文件规定地支关系、神煞、大运流年的报告写法。

## 地支关系

| 关系 | 现实翻译 | 报告边界 |
| --- | --- | --- |
| 合 | 关系牵引、资源绑定、议题被拉到一起 | 不直接等于好；可能是合作，也可能是黏连 |
| 冲 | 变化、迁移、对立、节奏被打断 | 不直接等于灾；要看冲到哪个宫位/十神和大运 |
| 刑 | 内耗、规则不顺、重复摩擦 | 避免恐吓，写成流程、边界和情绪管理 |
| 害 | 暗耗、误解、隐性不舒服 | 适合写沟通缺口和关系隐患 |
| 破 | 计划被拆、旧结构松动 | 适合写调整、拆解和重组 |
| 三合/六合 | 资源聚合、主题成局 | 要看是否化成可用结构，不能只看名词 |

## 神煞使用

- 神煞只能作为象意补充，不覆盖格局、十神、大运和现实校准。
- 桃花可写气质、社交吸引、审美和关系注意力，不直接断感情结果。
- 驿马可写移动、外部机会、换环境和跨区域资源，不直接断搬家。
- 华盖可写审美、孤独感、研究性和精神世界，不直接断孤僻或宗教。
- 天乙、文昌等吉神可写资源、学习、贵人和表达便利，但不能抵消主结构风险。

## 大运流年写法

每个年份建议按固定格式：

```text
盘面触发：大运/流年触发了什么十神、五行、地支关系。
现实倾向：可能落到事业、钱、关系、精力的哪类压力或机会。
可校准问题：过去是否出现对应经历。
建议动作：当年更适合怎么准备、收口或推进。
```

## 时间建议层

时间建议不是单独预测事件，而是把大运、流年、流月、流日拆成不同粒度的行动节奏：

| 层级 | 适合回答 | 不适合回答 |
| --- | --- | --- |
| 大运 | 这十年的长期主题、能力建设、关系和资源结构 | 某一天一定发生什么 |
| 流年 | 当年更该扩张、收口、换环境、做作品、谈合作还是修复基础 | 无条件吉凶、发财或分手承诺 |
| 流月 | 某几个月适合启动、谈资源、定框架、发布、复盘、观望或修正 | 用月份替代现实条件和准备程度 |
| 流日 | 已经确定事项后的短窗口，例如签约、会谈、发布、搬迁、面试 | 泛泛问“什么时候发财/脱单/翻身” |

写作顺序固定为：

```text
时间层级：大运 / 流年 / 流月 / 流日。
盘面触发：当前大运、流年流月流日和原局发生了什么关系。
适合动作：推进、签约、谈资源、复盘、修正、观望、休整中的哪一种。
前置条件：现实里必须已经具备什么，例如资料、报价、对方回复、合同草案、资金、人手或场地。
避坑提醒：这个窗口不适合做什么。
复盘指标：事后用什么观察是否有效。
```

边界：

- 流年和流月可以写趋势和准备，不写绝对结果。
- 流日只用于已经有明确事项的短窗口；没有事项时，先问清问题或改用六爻/小六壬。
- 时间建议要结合现实条件：客户、合同、资金、身体状态、团队准备、天气交通和政策限制都高于象意。
- 如果出生时间、真太阳时或时辰边界不稳，所有流月流日建议都要降级为待校准。

## flow_timing_report 写法

`flow_timing_report` 是流月流日行动节奏报告，主交付是 HTML，Markdown 只作源稿和审稿稿。

固定结构：

1. 一句话总判断。
2. 未来几段流月差异。
3. 重点日期。
4. 每日表。
5. 使用边界。

每日表必须至少包含：

```text
日期、流月、流日、今天就看这件事、现实动作、盘面触发。
```

禁止把每日建议写成抽象标签：

- 结构排雷。
- 资源谈判。
- 压力落地。
- 注意沟通。
- 适合推进。
- 谨慎合作。

必须写成具体动作：

- 别在火气里认领新锅。
- 把人情翻译成条款。
- 只跑小闭环，不搞大改革。
- 高冲突日，只谈事实。
- 谈钱谈资源，别谈感觉。
- 不要在群里临时拍大板。
- 没讲清的，不进执行月。

每份报告先列 case-specific 指纹。对 liujiang 这类项目经营场景，默认落到 OPC、Liujiang OS、Agent 项目包、团队训练、报价、合作、回款、ROI、对上汇报、不替别人兜底和小闭环试运行。其他 case 必须换成自己的真实场景，不能复用这些词假装个性化。

## 未来趋势边界

- 写趋势，不写绝对事件。
- 写准备，不写保证。
- 写风险管理，不写恐吓。
- 写“如果你已经在做某事，会更明显”，不要凭空制造事件。
