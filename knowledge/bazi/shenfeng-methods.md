# Bazi Shenfeng Methods

source_id: `SRC-BAZI-SHENFENG-TONGKAO`
depends_on: `knowledge/bazi/shenfeng-anchors.md`, `knowledge/bazi/structure-and-flow.md`, `knowledge/bazi/ten-gods.md`

这个文件把《神峰通考》相关方法从“术语锚点”推进到“报告可执行推理”。它仍然不是古文摘抄，也不是单句断命工具；任何判断必须先有脚本事实，再进入结构解释，最后落到现实建议。

## SF-M001 病药四步

- `source`: `SRC-BAZI-SHENFENG-TONGKAO`, `SRC-BAZI-PROJECT-SYNTHESIS`
- `rule`: 先找结构里的失衡点，再找能让结构恢复流通的调理点；不要把“缺什么”直接等同于“用什么”。
- `scope`: 日主强弱、月令气势、十神拥堵、五行寒暖燥湿、地支冲合刑害同时出现时。
- `limits`: 不用于医疗诊断，不写开运物件，不把单一五行当万能药。
- `modern_translation`: 现实里先问“卡在哪里”：是资源不足、授权不足、表达过载、规则太重、行动太急，还是边界太松。
- `report_usage`: 用“问题 -> 机制 -> 现实补法 -> 校准问题”四段写。例如先指出压力机制，再给流程、边界、休息、训练、合作方式等可执行建议。

## SF-M002 盖头截脚看表里不一致

- `source`: `SRC-BAZI-SHENFENG-TONGKAO`
- `rule`: 天干像外显能力和被看见的角色，地支像底层根气和现实承载；上下不一致时，要写“能表现”和“能落地”之间的落差。
- `scope`: 天干透出十神明显，但地支无根、被冲、被合走，或地支有根而天干不透时。
- `limits`: 不直接断好坏；盖头截脚只说明承接关系，不单独决定事业、财富或关系结果。
- `modern_translation`: 外面看起来能扛、能说、能做，但底层资源、团队、时间或情绪承载未必同步。
- `report_usage`: 在事业和合作章节写“你被期待承担什么”和“你实际需要什么支撑”，避免只夸能力不写代价。

## SF-M003 动静触发决定何时显化

- `source`: `SRC-BAZI-SHENFENG-TONGKAO`, `SRC-RUNTIME-LUNAR-PYTHON`
- `rule`: 原局里不是每个元素都同等活跃；透出、通根、冲合、刑害、大运流年引动后，某个议题才更容易进入现实。
- `scope`: 解释为什么某类问题不是一直发生，而是在某段大运、某年、某类合作环境中特别明显。
- `limits`: 触发不等于必然事件；必须保留现实条件和客户反馈入口。
- `modern_translation`: 盘里有倾向，不代表马上发生；被环境点燃时，才会从性格底色变成真实事项。
- `report_usage`: 流年段落按“原局伏笔 -> 大运/流年触发 -> 现实场景 -> 风险动作”写，不写绝对结果。

## SF-M004 枯旺损益写成系统状态

- `source`: `SRC-BAZI-SHENFENG-TONGKAO`
- `rule`: 枯、旺、弱、损等词只用于描述系统状态：过旺是过载，过枯是供给不足，受损是承载被破坏，偏弱是需要支援。
- `scope`: 五行或十神一边倒、季节气势明显、同类过多或某一链条断裂时。
- `limits`: 不把“病”写成身体疾病，不把“旺”简单写成好运，也不把“弱”写成无能。
- `modern_translation`: 过旺可能是用力过猛、控制过强、输出过度；过枯可能是资源稀缺、反馈不足、恢复不足。
- `report_usage`: 健康/精力章节只写生活节奏、恢复机制和压力管理；事业/财富章节写资源配置和流程承载。

## SF-M005 反例优先级

- `source`: `SRC-BAZI-SHENFENG-TONGKAO`, `SRC-BAZI-PROJECT-SYNTHESIS`
- `rule`: 如果脚本事实和神峰方法词冲突，以脚本事实、月令、十神主结构、大运流年为先；神峰术语只作解释框架。
- `scope`: 客户报告、综合命盘、流年分析和校准后的复盘。
- `limits`: 不因一个术语推翻整体结构，不为追求戏剧性制造断言。
- `modern_translation`: 好报告不是把术语堆满，而是说明一个机制为什么成立、哪里可能不成立。
- `report_usage`: 每个强判断前检查三件事：脚本事实是否支持、是否有反例、是否能落到现实动作。

## 付费报告落地模板

```text
结构事实：脚本显示的日主、月令、十神、地支关系、大运/流年触发。
神峰方法：这里属于病药、盖头截脚、动静触发或枯旺损益中的哪一种。
现实翻译：它在你的工作、钱、关系、精力中可能表现成什么。
反例边界：什么情况下这个判断会减弱或不成立。
行动建议：你现在可以做的流程、边界、节奏、沟通或资源调整。
校准问题：过去是否在类似年份/环境出现过对应模式。
```

## 禁用写法

- 禁止把“病药”写成身体疾病或治疗建议。
- 禁止把“缺某五行”直接写成“必须补某物”。
- 禁止把“盖头截脚”写成固定凶断。
- 禁止把“动静触发”写成某年必然发生某事。
- 禁止把古籍名当作权威压人；报告必须让读者看到现实机制。
