# Cost Accounting Notes

这不是报价表，也不是商业化建议。它只用于记录个人实验中的人工耗时、模型消耗、返工原因和工具成本，避免误判一个流程到底花了多少时间。

## 三行核算

每次完整测试都可以分三行看：

- `input value`：本次测试投入的样本、校准信息或反馈质量。
- `human cost = editing labor`：人工校准、写作、复核、沟通、修改时间。
- `compute = model/service cost`：模型、导出、存储、工具、平台等服务成本。

不要把人工成本和模型成本混成一个数，否则后面无法判断流程是不是真的轻量。

## 耗时估算

```text
total_cost_of_run = human_hours + compute_cost + rework_cost + review_cost
```

最危险的是“感觉很快”，实际每次完整长文都在人工润色、事实复核和返工里被吃掉。

## 测试层级

### 体验版

- 低门槛样本测试。
- 目标：验证“结构化命盘 + 可读解释”是否能被读懂。
- 控制：不做 DOCX/PDF 精排，不做长流年，不承诺修订。

### 标准版

- 完整流程测试。
- 目标：验证事实档案、长文、简洁版、手机阅读器和复盘闭环。
- 控制：只给一轮轻修；资料错误导致重排需另计。

### 深度版

- 深度校准测试。
- 目标：验证补充经历后，报告能否更准确地区分事实、推断和建议。
- 控制：必须先收校准信息；不接高风险决策。

### 专题版

- 单点问题测试。
- 主题：事业、财富、感情、合伙、年度节奏。
- 控制：问题必须明确；六爻/小六壬只回答当下问题，不覆盖本命盘。

## 返工规则

免费修订只覆盖：

- 错别字。
- 语气调整。
- 客户补充校准后的小范围措辞修正。

不免费覆盖：

- 客户出生时间、性别、出生地给错导致重排。
- 从标准版扩展成深度专题。
- 要求删除边界声明或改成绝对预测。
- 要求写医疗、法律、投资等高风险结论。

## 要记录的复盘指标

- 人工耗时。
- 模型/工具成本。
- 返工次数。
- 读者反馈。
- 哪些段落最容易被截图转发。
- 哪些问题最容易导致纠纷。
