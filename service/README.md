# Xuanxue Console Workflow Notes

这个目录记录 `xuanxue-console` 从“能生成文本”到“能稳定复查、打包和复盘”的流程笔记。它不是商业化指南，也不鼓励把项目简单包装成付费算命服务。核心原则不变：计算事实由脚本生成，解释长文由人和模型基于事实写作，所有输出都要经过验收，不把玄学报告包装成医疗、法律、投资或婚恋决定。

## 实验输出层级

### 体验版

- 适合：朋友内测、个人实验、流程验证。
- 输入：出生信息、性别、出生地、1-2 个关心主题。
- 输出：结构化排盘摘要、3000-5000 字短报告、3-5 个校准问题。
- 不承诺：完整流年长文、DOCX/PDF 精排、二次修订。

### 标准版

- 适合：完整测试本项目的主流程。
- 输入：出生信息、性别、出生地、现居地、MBTI 可选、关心主题、过去校准信息；原始资料统一放外部 `<EXTERNAL_ROOT>/inputs/<case_id>`。
- 输出：reader-rich 长文，约 18000-24000 字符；第 01 章采用判断型摘要；Markdown、DOCX、PDF、干净 ZIP；后台 manifest 和结构化 JSON 留在外部 `<RUN_DIR>` 作内部证据。
- 验收：`validate_longform_report.py --profile reader-rich`、`audit_case_isolation.py`、DOCX/PDF 可读性检查、`finalize_case.py`、`audit_project_hygiene.py`。

### 深度版

- 适合：愿意补充经历、希望测试校准能力的人。
- 输入：标准版资料，加 15-30 分钟问答或文字校准。
- 输出：标准版交付，加一轮校准修订、专题建议页、未来 12-24 个月行动清单。
- 边界：不做确定性承诺，不替代专业咨询。

### 专题版

- 适合：事业、感情、合伙、年度计划、合盘等单点问题。
- 输出：出生盘事实 + 指定主题深挖；必要时用六爻/小六壬作为问事快照，但不覆盖本命盘。
- 风险：专题越窄，越要提前写清楚“回答范围”和“不能回答什么”。

## 设计目标

- 不是纯聊天：先用脚本排盘，后写解释。
- 不是模板套话：每份报告要引用本 case 的四柱、紫微宫位、西占落座和校准年份。
- 不是吓人话术：只写趋势、性格、关系和行动建议，不写死亡、疾病诊断或绝对命运。
- 适合转发：读者版正文不暴露脚本路径、JSON 路径、执行命令和内部验收说明。
- 可复核：外部 run manifest 保存输入口径、命令、产物、验证结果和已知边界。
- 不串案：项目仓库不保存客户产物；每个 case 有独立 `<RUN_DIR>`，交付前跑 case isolation 和 project hygiene。

## 最小完整流程

1. 用 `service/client-intake-form.md` 收资料。
2. 用 `scripts/create_case_workspace.py` 创建外部 workspace。
3. 按 `service/production-sop.md` 生成结构化事实和 reader-rich 长文。
4. 按 `service/quality-gate.md` 验收。
5. 交付 `<RUN_DIR>/delivery/` 里的丰富 PDF、简洁 PDF 和对应手机阅读器；ZIP 只在用户明确需要时附加。
6. 用 4-8 个校准问题收反馈；反馈只修正文案，不改脚本事实。

## 风险边界

- 所有页面都要写明：传统文化与自我观察参考，不替代医疗、法律、投资、婚恋决定或人生重大决策。
- 不接高风险诉求：疾病断生死、投资买卖点、离婚/结婚决定、法律胜负、强诱导付费。
- 客户隐私默认不进入公开案例；公开案例必须去标识化。
- 成本核算只用于理解工具消耗和人工耗时，不代表鼓励商业化。
